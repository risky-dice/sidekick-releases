# HWP/HWPX 작업 안전 체크리스트

HWP, HWPX 문서를 수정할 때 적용한다.

## 작업 전
- 한글 프로그램이 파일을 잡고 있으면 닫거나 백업본으로 작업한다.
- HWP 원본을 직접 덮어쓰기 전에 백업을 만든다.
- 사이드카 워크스페이스 파일과 바탕화면 원본 중 어느 파일이 기준인지 확인한다.

## 수정 방식
- 가능하면 HWP를 HWPX로 변환한 뒤 구조를 확인하고 다시 HWP로 저장한다.
- 표 셀 수정은 행/열 위치를 검증한 뒤 반영한다.
- `SetTextFile`처럼 인코딩 선택 창을 띄우는 방식은 피한다.
- 한글 문자열을 스크립트에 넣을 때 UTF-8 보존 여부를 확인한다.

## 검증
- 수정 후 HWP를 다시 HWPX로 변환해 텍스트를 추출한다.
- 한글이 `?`로 깨지지 않았는지 확인한다.
- 표의 행/열 순서가 밀리지 않았는지 확인한다.
- 최종본과 사이드카 워크스페이스 사본이 같은지 확인한다.
