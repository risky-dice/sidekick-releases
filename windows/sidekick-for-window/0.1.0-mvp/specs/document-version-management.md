# Sidekick 문서 버전 관리 설계

## 목표

사용자가 보는 문서는 항상 하나로 유지하고, Sidekick 내부에서만 자동 버전과 변경 기록을 관리한다.

## 기본 원칙

- 바탕화면 또는 사용자가 지정한 위치의 문서는 그대로 덮어쓴다.
- 덮어쓰기 직전 현재 파일을 내부 vault에 스냅샷으로 저장한다.
- 새 수정본은 임시 파일로 만든 뒤 열기/텍스트 추출 검증을 통과하면 원본 위치에 덮어쓴다.
- 사용자가 직접 볼 필요 없는 백업 파일은 바탕화면에 만들지 않는다.
- 문서 파일 원본은 로컬에 보관한다.
- Supabase 같은 외부 DB에는 파일 본문이 아니라 메타데이터만 저장한다.

## 로컬 폴더 구조

```text
sidekick
└─ document-vault
   └─ {document_id}
      ├─ original
      │  └─ original.hwp
      ├─ current
      │  └─ current.hwp
      ├─ versions
      │  ├─ v001_original_20260623_0910.hwp
      │  ├─ v002_before_budget_edit_20260623_1014.hwp
      │  └─ v003_before_submit_20260623_1121.hwp
      ├─ exports
      │  └─ submit_20260623_안양문화고.hwp
      └─ history.md
```

## 덮어쓰기 절차

1. 현재 원본 파일의 존재 여부를 확인한다.
2. 원본 파일을 `versions`에 새 버전으로 복사한다.
3. 수정본을 임시 파일로 생성한다.
4. 수정본이 정상적으로 열리는지 확인한다.
5. 가능하면 HWPX 변환 후 주요 텍스트를 추출해 한글 깨짐을 확인한다.
6. 검증에 통과하면 원본 위치에 덮어쓴다.
7. `current/current.hwp`에도 같은 파일을 복사한다.
8. `history.md`에 변경 내용을 남긴다.
9. Supabase 연동이 켜져 있으면 메타데이터만 업로드한다.

## 자동 정리 정책

- 최근 버전 10개는 항상 보관한다.
- 사용자가 보호 표시한 버전은 삭제하지 않는다.
- 하루가 지난 자동 버전은 날짜별 대표 버전 1개만 남긴다.
- 한 달이 지난 자동 버전은 월별 대표 버전 1개만 남긴다.
- 문서 1개당 vault 크기가 300MB를 넘으면 오래된 자동 버전부터 정리한다.
- 전체 vault 크기가 2GB를 넘으면 정리 안내를 띄운다.

## 보호 버전

아래 버전은 자동 삭제하지 않는다.

- 최초 원본
- 제출본
- 사용자가 직접 보호 표시한 버전
- 큰 구조 변경 직전 버전
- 공문 제출 직전 버전

## 권장 기본값

- 저장 방식: 겉으로는 덮어쓰기, 내부적으로는 자동 버전 저장
- 자동 백업: 켜짐
- 자동 정리: 켜짐
- Supabase 연동: 선택
- 파일 원본 업로드: 꺼짐
